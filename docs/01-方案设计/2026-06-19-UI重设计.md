# DeployMaster 专业级布局设计方案

## 一、设计目标

创建一个专业级、工业级的批量运维工具界面，具备以下特点：

1. **灵活可调**：使用 QSplitter 实现各区域的动态调整
2. **信息分层**：合理组织配置、操作、监控三类信息
3. **高效操作**：常用操作一键可达，减少鼠标移动距离
4. **清晰反馈**：实时状态展示，操作结果一目了然

## 二、布局结构

### 2.2 顶部配置栏设计

**当前问题**：IP列表和认证区域比例不合理，存在信息重复

**改进方案**：
- 去掉"默认远程路径"（与FTP设置重复）
- IP列表:认证区域比例调整为 3:1
- 认证区域仅保留用户名和密码
- 增加IP列表高度至60px，便于查看更多IP

### 2.3 工作区布局

**左侧工作区**（主区域）：
- Tab页切换（批量部署、日志查询、批量命令等）
- 每个Tab内部独立布局
- 文件上传区域增加最小高度至150px
- FTP设置区域与操作按钮合并为一行

**右侧面板**（辅助区域）：
- 远端预览：使用 QSplitter 动态调整宽度（默认约占 25%，即 splitter_main setSizes 600:200）
- 状态信息：可折叠面板（待实现）
- 快捷操作：常用命令快捷按钮（待实现）

### 2.4 底部日志区

- 使用 QSplitter 实现高度动态调整
- 默认占窗口高度的 25%（splitter_log setSizes 375:125）
- 支持拖动分割线调整

## 三、具体改动

### 3.1 UI 文件改动（DeployMaster.ui）

1. **顶部配置栏**：
   - 删除 `txt_defaultRemotePath` 和 `label_defaultRemotePath`
   - 调整水平布局比例为 3:1
   - IP列表高度从50px调整为60px
   - 认证区域高度从80px调整为60px

2. **工作区布局**：
   - 使用 QHBoxLayout 替代 QGridLayout
   - 添加 QSplitter 分隔左右区域
   - 左侧：TabWidget（约占75%宽度）
   - 右侧：远端预览面板（使用 QSplitter 动态调整，无固定宽度）

3. **底部日志区**：
   - 使用 QSplitter 分隔工作区和日志区
   - 日志区默认高度比例为 25%

### 3.2 代码改动

1. **DeployMaster.cpp**：
   - 移除对 `txt_defaultRemotePath` 的引用
   - 调整日志区域大小策略
   - 添加分割线拖动事件处理

2. **样式表**：
   - 优化 QSplitter 样式
   - 调整面板边框和间距

## 四、预期效果

1. **空间利用率提升**：去掉重复信息，减少视觉噪音
2. **操作效率提升**：合理的布局比例，常用操作更易访问
3. **用户体验提升**：灵活调整各区域大小，适应不同屏幕尺寸
4. **专业感提升**：清晰的信息分层，符合工业级工具标准

## 五、风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 布局改动可能导致部分功能失效 | 中 | 逐项测试每个Tab页功能 |
| 用户习惯改变 | 低 | 保持核心操作位置不变 |
| 多分辨率适配问题 | 低 | 使用相对布局和比例 |
