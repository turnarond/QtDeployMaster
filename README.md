# DeployMaster - SylixOS PLC 部署与运维平台

DeployMaster 是一款基于 Qt 6 开发的工业级 SylixOS PLC 部署与运维管理平台，专为批量设备管理、测试验证和日常运维设计。它集成了文件部署、命令执行、日志管理、Modbus 测试、OPC UA 通信和诊断分析等多种功能，为 SylixOS 工业自动化系统的全生命周期管理提供全面的解决方案。

## 产品概述

### 背景与定位

DeployMaster 与 PLCBasicConfigurator 共同构成 SylixOS PLC 设备的完整工具链：
- **DeployMaster**：负责设备部署、测试验证和批量运维
- **PLCBasicConfigurator**：负责设备配置管理

两者协同工作，为 SylixOS PLC 设备提供从配置、部署、测试到运维的一站式服务。

### 核心价值

- **效率提升**：批量操作支持，将管理效率提升10倍以上
- **故障定位**：集成诊断工具，快速定位主备冲突等复杂问题
- **稳定可靠**：工业级设计，支持长时间稳定运行
- **易于使用**：直观的图形界面，降低操作门槛

## 功能特性

### 1. 批量部署模块

- **文件和文件夹上传**
  - 支持单文件或整个文件夹批量上传
  - 拖拽式文件添加，操作便捷
  - 支持多种文件类型筛选（可执行文件、配置文件、脚本等）

- **远程路径管理**
  - 可自定义远程部署路径
  - 支持部署前清空远程目录
  - 远程文件结构预览和浏览

- **部署后操作**
  - 支持部署完成后自动重启设备
  - 实时显示部署进度和状态
  - 详细的部署日志记录

### 2. 批量命令模块

- **Telnet 命令执行**
  - 在多个设备上并行执行 Telnet 命令
  - 支持多行命令批量执行
  - 可配置命令超时时间

- **实时状态监控**
  - 实时显示每个设备的执行状态
  - 统计命令执行耗时
  - 错误信息实时反馈

- **结果管理**
  - 收集和缓存每个设备的命令输出
  - 双击设备查看详细的命令执行输出
  - 支持导出所有执行结果到文件

### 3. 日志查询模块

- **日志文件浏览**
  - 查看远程设备上的日志文件列表
  - 显示文件名、大小、修改时间等信息
  - 支持按条件筛选日志文件

- **日志下载管理**
  - 支持下载选中的日志文件
  - 支持批量下载全部日志文件
  - 本地日志文件结构化存储

### 4. Modbus 集群测试模块

- **批量寄存器读取**
  - 同时读取多个设备的 Modbus 寄存器值
  - 支持多种寄存器类型（保持寄存器、输入寄存器等）
  - 可配置从站 ID、起始地址和读取长度

- **数据对比分析**
  - 对比不同设备间的寄存器值
  - 高亮显示不一致的数据
  - 确保集群数据一致性

- **寄存器写入功能**
  - 支持向选定设备写入相同值
  - 支持写入随机值进行测试
  - 支持写入特定值进行验证

- **实时监控**
  - 定时自动刷新功能
  - 实时监控设备状态变化
  - 操作耗时统计和性能评估

### 5. OPC UA 客户端模块

- **服务器连接管理**
  - 支持连接到 OPC UA 服务器
  - 多种安全策略支持
  - 连接状态实时显示

- **节点浏览与操作**
  - 浏览服务器中的 OPC UA 节点树
  - 读取和写入节点值
  - 支持订阅节点值的变化

### 6. WebSocket 通信模块

- **实时通信**
  - 基于 WebSocket 的双向实时通信
  - 支持消息的发送和接收
  - 连接状态监控和自动重连

### 7. 诊断分析模块（基于 VSOA 协议）

- **设备连接管理**
  - 基于 VSOA (Vehicle Service-Oriented Architecture) 协议
  - 自动检测设备连接状态
  - 支持多设备批量管理

- **一键诊断采集**
  - 触发远程设备诊断采集
  - 支持配置采集参数（日志行数、Dump 文件等）
  - 实时显示任务进度和状态

- **智能下载管理**
  - 自动下载诊断包
  - 支持断点续传
  - 多线程异步下载，不阻塞 UI

- **诊断包分析**
  - 内置解压引擎，直接读取 .diag 包
  - 多视图日志查看器
  - 关键字自动高亮（Error、Fail、Master、Standby 等）

- **主备对比分析**
  - 双栏布局对比主备设备
  - 进程/容器状态自动对比
  - 配置文件差异对比
  - 异常行高亮显示

- **统一时间轴视图**
  - 从多机日志中提取时间戳事件
  - 在公共时间轴上混合展示
  - 帮助分析事件因果关系

## 安装部署

### 方法一：使用预编译二进制文件（推荐）

1. **下载程序**
   - 从项目发布页面下载最新的发布版本
   - 选择与您操作系统匹配的版本

2. **解压安装**
   ```powershell
   # Windows
   Expand-Archive DeployMaster-v1.0.0-win64.zip -DestinationPath C:\DeployMaster
   
   # Linux
   tar -xzf DeployMaster-v1.0.0-linux64.tar.gz -C /opt
   ```

3. **运行程序**
   ```powershell
   # Windows
   C:\DeployMaster\DeployMaster.exe
   
   # Linux
   /opt/DeployMaster/DeployMaster
   ```

### 方法二：从源码编译

#### 前置准备

1. **安装 Qt 6.10.1**
   - 从 [Qt 官网](https://www.qt.io/download) 下载 Qt 6.10.1
   - 安装时确保选择以下组件：
     - Qt 6.10.1 (MSVC 2022 64-bit 或 MinGW 64-bit)
     - Qt SerialBus 模块
     - Qt WebSockets 模块

2. **安装编译工具**
   - **Windows**：Visual Studio 2022（包含 CMake 工具）
   - **Linux**：build-essential、cmake、git

#### 编译步骤

1. **克隆代码仓库**
   ```bash
   git clone https://github.com/your-org/DeployMaster.git
   cd DeployMaster/DeployMaster
   ```

2. **创建构建目录**
   ```bash
   mkdir build
   cd build
   ```

3. **配置 CMake**
   ```powershell
   # Windows (PowerShell)
   cmake .. -DCMAKE_PREFIX_PATH="C:\Qt\6.10.1\msvc2022_64"
   
   # Linux
   cmake .. -DCMAKE_PREFIX_PATH="/opt/Qt/6.10.1/gcc_64"
   ```

4. **编译项目**
   ```bash
   # Windows
   cmake --build . --config Release
   
   # Linux
   cmake --build .
   ```

5. **运行程序**
   ```bash
   # Windows
   .\Release\DeployMaster.exe
   
   # Linux
   ./DeployMaster
   ```

## 快速入门

### 第一次使用

1. **启动程序**
   - 双击运行 `DeployMaster.exe`
   - 程序启动后会显示主界面

2. **配置设备列表**
   - 在主界面顶部的"目标 IP"文本框中输入设备 IP 地址
   - 每行一个 IP，支持格式：`192.168.1.100` 或 `192.168.1.100:21`
   - 示例：
     ```
     192.168.1.100
     192.168.1.101
     192.168.1.102
     ```

3. **设置认证信息**
   - 输入 FTP 和 Telnet 登录的用户名和密码
   - 默认用户名：`root`，密码：`sylixos`

4. **配置远程路径**
   - 设置默认的远程部署路径
   - 默认为：`/apps/`

### 第一次部署

1. **添加部署文件**
   - 点击"添加文件"按钮选择要部署的文件
   - 或直接拖拽文件到上传列表区域
   - 也可以点击"添加文件夹"选择整个目录

2. **配置部署选项**
   - 勾选"部署前清空远程目录"（可选）
   - 勾选"部署后重启设备"（可选）
   - 确认远程路径设置正确

3. **开始部署**
   - 点击"开始部署"按钮
   - 观察日志窗口中的部署进度
   - 等待部署完成

## 功能模块详解

### 批量部署模块详解

#### 功能说明

批量部署模块用于将本地文件或文件夹上传到多台 SylixOS PLC 设备，支持部署后自动重启。

#### 操作步骤

1. **准备部署内容**
   - 确保本地文件已准备就绪
   - 确认所有设备网络连通

2. **添加上传项**
   ```
   方式一：点击按钮添加
   - 点击"添加文件" → 选择文件 → 确认
   - 点击"添加文件夹" → 选择目录 → 确认
   
   方式二：拖拽添加
   - 直接从文件管理器拖拽文件/文件夹到上传列表
   ```

3. **配置选项**
   - **远程路径**：文件上传到设备的目标路径
   - **清空远程目录**：上传前清空目标目录（慎用）
   - **重启设备**：部署完成后自动重启设备

4. **执行部署**
   - 点击"开始部署"
   - 系统将并行上传到所有设备
   - 实时查看部署日志和状态

#### 注意事项

- 清空远程目录操作不可逆，请谨慎使用
- 大文件上传可能需要较长时间，请耐心等待
- 部署过程中请勿关闭程序或断开网络

### 批量命令模块详解

#### 功能说明

通过 Telnet 协议在多台设备上并行执行命令，支持实时查看执行状态和输出。

#### 操作步骤

1. **准备命令**
   - 在左侧命令编辑区输入要执行的命令
   - 每行一条命令，支持多行命令
   - 示例：
     ```bash
     ls -la /apps
     ps aux
     ifconfig
     ```

2. **配置选项**
   - 设置命令超时时间（默认 30 秒）
   - 确认目标设备列表正确

3. **执行命令**
   - 点击"执行命令"按钮
   - 右侧设备树显示每个设备的执行状态
   - 状态包括：等待中、执行中、成功、失败

4. **查看结果**
   - 双击设备节点查看详细的命令输出
   - 点击"导出结果"保存所有执行结果

#### 常用命令示例

```bash
# 查看系统信息
uname -a
cat /proc/version

# 查看进程
ps aux
top -b -n 1

# 查看网络
ifconfig
netstat -an

# 查看磁盘
df -h
du -sh /apps

# 查看日志
tail -100 /var/log/messages
dmesg | tail -50
```

### Modbus 集群测试模块详解

#### 功能说明

对多台 Modbus TCP 设备进行寄存器读写测试，支持数据对比和定时刷新。

#### 操作步骤

1. **配置设备列表**
   - 在设备列表中输入 Modbus 设备地址
   - 格式：`IP:Port` 或 `IP:Port:SlaveID`
   - 示例：
     ```
     192.168.1.100:502
     192.168.1.101:502:1
     192.168.1.102:502:2
     ```

2. **设置 Modbus 参数**
   - **寄存器类型**：保持寄存器、输入寄存器、线圈、离散输入
   - **从站 ID**：默认 1（如设备地址中已包含则忽略）
   - **起始地址**：寄存器起始地址（十进制）
   - **读取长度**：要读取的寄存器数量

3. **读取寄存器**
   - 点击"读取并对比"按钮
   - 系统并行读取所有设备的寄存器
   - 结果表格中显示各设备的寄存器值
   - 不一致的值会高亮显示

4. **写入寄存器**
   - 选择要写入的设备（在设备列表中勾选）
   - 选择写入方式：
     - **相同值**：所有寄存器写入相同的值
     - **随机值**：每个寄存器写入随机值
     - **特定值**：按顺序写入指定的值
   - 点击"写入选定设备"执行写入

5. **定时刷新**
   - 勾选"定时刷新"启用自动刷新
   - 设置刷新间隔（秒）
   - 系统会自动定期读取寄存器并更新表格

#### 数据对比说明

- 所有设备的值相同：显示正常颜色
- 存在不同值：不一致的单元格标红
- 读取失败：显示"错误"或"超时"

### 诊断分析模块详解

#### 功能说明

基于 VSOA 协议的 SylixOS PLC 诊断工具，用于快速定位主备冲突等复杂故障。

#### 操作步骤

1. **添加设备**
   - 点击"添加设备"按钮
   - 输入设备 IP 地址和名称
   - 设备会自动尝试连接并显示状态

2. **启动诊断**
   - 勾选要诊断的设备
   - 配置采集参数（可选）：
     - 日志行数：默认 1000 行
     - 包含 Dump 文件：是/否
   - 点击"开始诊断"按钮

3. **监控进度**
   - 任务表格显示每个设备的诊断进度
   - 状态包括：等待中、采集中、下载中、完成、失败

4. **查看诊断包**
   - 诊断完成后，诊断包自动下载到本地
   - 在左侧日志树中选择要查看的诊断包
   - 右侧显示诊断包内容

5. **主备对比**
   - 点击"打开对比视图"
   - 左侧加载主机诊断包，右侧加载备机诊断包
   - 选择对比类型：进程对比、配置对比、日志对比
   - 系统自动高亮显示差异

#### 诊断包内容

诊断包（.diag 文件）通常包含：

| 文件 | 说明 |
|------|------|
| proc.txt | 进程列表快照 |
| ecs.json | 容器状态信息 |
| dmesg.txt | 内核日志 |
| messages | 系统日志 |
| config/ | 配置文件目录 |
| dump/ | 崩溃转储文件（如启用） |

## 工作流程

### 典型部署工作流

```
1. 准备阶段
   ├─ 确认设备网络连通
   ├─ 准备部署文件
   └─ 配置 DeployMaster

2. 连接测试
   ├─ 输入设备 IP 列表
   ├─ 设置认证信息
   └─ 测试 FTP/Telnet 连接

3. 执行部署
   ├─ 添加部署文件/文件夹
   ├─ 配置部署选项
   ├─ 开始批量部署
   └─ 监控部署进度

4. 验证阶段
   ├─ 检查部署日志
   ├─ 批量执行验证命令
   ├─ Modbus 寄存器验证（如需要）
   └─ 确认设备运行正常

5. 完成
   ├─ 导出部署报告
   ├─ 保存配置
   └─ 记录操作日志
```

### 故障诊断工作流

```
1. 故障报警
   ├─ 发现设备异常
   ├─ 收集初步信息
   └─ 启动 DeployMaster

2. 连接设备
   ├─ 添加故障设备
   ├─ 检查连接状态
   └─ 确认设备在线

3. 采集诊断
   ├─ 配置采集参数
   ├─ 启动一键诊断
   ├─ 等待诊断包生成
   └─ 自动下载诊断包

4. 分析诊断
   ├─ 打开诊断包
   ├─ 查看系统日志
   ├─ 检查进程状态
   ├─ 主备对比分析（如适用）
   └─ 时间轴事件分析

5. 定位问题
   ├─ 识别错误信息
   ├─ 分析根因
   └─ 制定修复方案

6. 修复验证
   ├─ 执行修复操作
   ├─ 再次采集诊断
   ├─ 对比前后状态
   └─ 确认问题解决
```

## 常见问题

### 部署相关

**Q: 部署失败怎么办？**

A: 请按以下步骤排查：
1. 检查目标 IP 地址是否正确，设备是否在线
2. 确认用户名和密码是否正确
3. 检查网络连接是否正常，尝试 ping 设备
4. 查看日志窗口中的详细错误信息
5. 确认 FTP 服务在设备上正常运行（端口 21）

**Q: 大文件上传很慢怎么办？**

A: 优化建议：
1. 确保网络连接为千兆以太网
2. 关闭不必要的网络应用
3. 考虑压缩大文件后再上传
4. 分批上传，避免单次上传过多文件

**Q: 部署后设备无法启动怎么办？**

A: 应急处理：
1. 通过 Telnet 登录设备检查状态
2. 查看系统日志找出问题
3. 如有备份，恢复之前的版本
4. 必要时进入单用户模式修复

### 命令执行相关

**Q: 命令执行超时怎么办？**

A: 解决方案：
1. 在批量命令界面增加命令超时时间设置
2. 检查网络连接质量和延迟
3. 确认设备是否正常运行，负载是否过高
4. 对于长时间运行的命令，考虑改为后台执行

**Q: 某些设备命令执行失败怎么办？**

A: 排查步骤：
1. 检查该设备是否在线
2. 确认 Telnet 服务是否正常运行
3. 验证用户名和密码是否正确
4. 查看该设备的详细错误输出
5. 尝试手动 Telnet 登录测试

### Modbus 测试相关

**Q: 无法连接到 Modbus 设备怎么办？**

A: 检查清单：
1. 确认 Modbus TCP 端口（默认 502）是否正确
2. 验证从站 ID 设置是否正确
3. 确认设备是否支持 Modbus TCP 协议
4. 检查网络连接和防火墙设置
5. 使用其他 Modbus 工具测试连接

**Q: 读取的数据不对怎么办？**

A: 可能原因：
1. 寄存器地址设置错误（注意有些设备从 0 开始，有些从 1 开始）
2. 寄存器类型选择错误（保持寄存器 vs 输入寄存器）
3. 数据大小端格式不匹配
4. 设备端数据正在更新中

### 诊断相关

**Q: VSOA 连接失败怎么办？**

A: 排查步骤：
1. 确认设备上的诊断服务是否正常运行
2. 检查 VSOA 协议端口是否可访问
3. 验证网络连接是否正常
4. 查看设备端的服务日志

**Q: 诊断包下载很慢怎么办？**

A: 优化建议：
1. 减少采集的日志行数
2. 不包含 Dump 文件（如不需要）
3. 确保网络带宽充足
4. 考虑单独下载，避免同时下载多台设备

## 注意事项

### 安全注意事项

1. **密码管理**
   - 程序不会保存密码，每次使用需重新输入
   - 不要在公共场所或共享电脑上使用
   - 使用完毕后请及时关闭程序

2. **网络安全**
   - 建议在受控的局域网环境中使用
   - 避免在公网直接暴露设备管理端口
   - 考虑使用 VPN 进行远程访问

3. **操作安全**
   - 清空远程目录操作前请务必备份
   - 批量重启设备前请确认业务影响
   - 重要操作建议先在单台设备上测试

### 使用注意事项

1. **设备管理**
   - 定期更新设备列表，移除离线设备
   - 为设备设置有意义的名称，便于识别
   - 分组管理大量设备，避免列表过长

2. **文件管理**
   - 部署前验证文件的完整性和正确性
   - 使用版本控制管理部署文件
   - 保留历史版本，便于回滚

3. **日志管理**
   - 定期清理本地日志文件，避免占用过多磁盘空间
   - 重要操作日志建议导出保存
   - 建立日志归档策略

4. **性能优化**
   - 并发操作数量适中，避免过多并发导致网络拥塞
   - 大文件传输建议在网络负载较低时进行
   - 定时刷新间隔不宜过短，避免给设备造成压力

### 维护建议

1. **程序更新**
   - 关注项目发布页面，及时更新到最新版本
   - 更新前备份配置和重要数据
   - 新版本先在测试环境验证

2. **问题反馈**
   - 遇到问题先查看常见问题解答
   - 收集详细的错误信息和复现步骤
   - 通过 Issue 跟踪系统反馈问题

3. **最佳实践**
   - 建立标准操作流程（SOP）
   - 对操作人员进行培训
   - 定期进行应急演练
